---
title: "Get started with Orceum"
description: "Build and register a working Orceum app in under 10 minutes. This guide walks through a simple Todo Manager with API key auth and three actions."
---

This guide walks you through building a minimal Orceum app from scratch. You'll write an HTTP endpoint, register it with the platform, install it as a user, and verify it works end-to-end. The example is a **Todo Manager** — a Native HTTP app with API key auth and three actions: `todo.create`, `todo.list`, and `todo.complete`.

<Steps>
  <Step title="Write your endpoint">
    Create a single HTTP endpoint. Orceum will `POST` to it with `{ event, event_data, timestamp }`.

    ```python app.py
    from fastapi import FastAPI, Header

    app = FastAPI()
    todos = {}

    @app.post("/orceum/actions")
    async def handle(
        request,
        x_orceum_installation_id: str = Header(...),
    ):
        body = await request.json()
        event = body["event"]
        params = body["event_data"]

        if event == "todo.create":
            tid = str(len(todos) + 1)
            todos[tid] = {"id": tid, "title": params["title"], "done": False}
            return {"status": "success", "data": {"todo_id": tid}}

        if event == "todo.list":
            return {"status": "success", "data": {"todos": list(todos.values())}}

        if event == "todo.complete":
            todo = todos.get(params["todo_id"])
            if not todo:
                return {"status": "error", "error": {"message": "Not found"}}
            todo["done"] = True
            return {"status": "success", "data": {"message": f"Done: {todo['title']}"}}

        return {"status": "error", "error": {"message": f"Unknown event: {event}"}}
    ```

    <Note>
    `X-Orceum-Installation-Id` is sent on **every** call. Use it to identify which user triggered the action.
    </Note>
  </Step>

  <Step title="Register your app">
    Make a single API call to register your endpoint, auth config, and manifest. Orceum uses the manifest descriptions to understand what your app can do — the AI reads them to decide when and how to call you.

    ```bash
    curl -X POST https://api.orceum.com/v1/apps/ \
      -H "Authorization: Bearer YOUR_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "Todo Manager",
        "description": "Create, list, and manage todo items",
        "version": "1.0.0",
        "app_url": "https://your-app.example.com/orceum/actions",
        "auth_type": "API_KEY",
        "manifest": {
          "actions": [
            {
              "event": "todo.create",
              "description": "Create a new todo item for the user",
              "parameters": [
                { "name": "title", "type": "string", "description": "The todo title", "required": true }
              ]
            },
            {
              "event": "todo.list",
              "description": "List all todo items for the user",
              "parameters": []
            },
            {
              "event": "todo.complete",
              "description": "Mark a todo item as complete",
              "parameters": [
                { "name": "todo_id", "type": "string", "description": "ID of the todo to complete", "required": true }
              ]
            }
          ]
        },
        "config": {
          "auth_config": {
            "auth_header_key": "X-API-Key",
            "authentication_scheme": ""
          }
        }
      }'
    ```

    You'll get back an `app_id`. Save it — you'll need it in the next step.
  </Step>

  <Step title="Install the app as a user">
    Install the app against a user account:

    ```bash
    curl -X POST https://api.orceum.com/v1/apps/{app_id}/install \
      -H "Authorization: Bearer USER_TOKEN"
    ```

    Because `auth_type` is `API_KEY`, Orceum returns `next_action: "COLLECT_API_KEY"`. Submit the key using the `installation_id` from the response:

    ```bash
    curl -X POST https://api.orceum.com/v1/apps/installations/{installation_id}/api-key \
      -H "Authorization: Bearer USER_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{ "api_key": "my-secret-key" }'
    ```

    Orceum encrypts the key and injects it into every future call to your endpoint.
  </Step>

  <Step title="Test it">
    Say to Orceum: **"Create a todo: Buy groceries"**

    Orceum calls your endpoint:

    ```json
    POST /orceum/actions
    X-Orceum-Installation-Id: f47ac10b-...
    X-API-Key: my-secret-key

    {
      "event": "todo.create",
      "event_data": { "title": "Buy groceries" },
      "timestamp": "2026-04-15T10:30:00.000000+00:00"
    }
    ```

    Your app returns:

    ```json
    { "status": "success", "data": { "todo_id": "1" } }
    ```

    The AI tells the user: *"Done! Created your todo: Buy groceries (ID: 1)"*
  </Step>
</Steps>
